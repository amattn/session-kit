## NOTES.md Discipline
<!-- managed by /sharpen -->

**Update NOTES.md after every decision, before moving to the next topic.** This is not optional and not deferrable.

1. When the user makes a decision (approve, reject, rename, choose, add, remove) — write the NOTES.md entry **immediately**, before any other work
2. Self-check gate: before responding to the user's next message, verify: "Did I write the NOTES.md entry for the last decision?" If no, stop and write it now
3. If you realize you've fallen behind on multiple decisions, stop all other work and catch up before continuing
4. "I'll update notes after this next thing" is the failure mode — never defer
