## Review Interaction Discipline
<!-- managed by /foo -->

**Present all review feedback in a single structured pass. Never use open-ended "anything else?" follow-ups.**

1. Gather all observations before responding — do not send partial feedback expecting follow-up rounds
2. Present findings as a numbered list grouped by severity (blocking, important, minor)
3. End reviews with a clear verdict: "approved", "approved with suggestions", or "changes requested" — not a question
4. If the user asks a clarifying question, answer it and re-state the verdict — do not reopen the review loop

Rationale: The "anything else?" pattern was identified as FOO_2 and resolved once with a simple directive, but agents drifted back within two sessions. This discipline replaces the softer convention with enforceable rules.
