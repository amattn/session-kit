# CLAUDE.md

## Project

inventory-api — REST API for managing inventory.

## NOTES.md Discipline

**Update NOTES.md after every decision, before moving to the next topic.**

## Process Improvement

This project uses `PROCESS.md` to track process observations and formalize them into disciplines.

- **When you notice a friction point, recurring mistake, or helpful pattern**: add an observation to `PROCESS.md` (status: `observed`, tally: 1).
- **When an observation recurs**: increment its tally and update status to `validated` when there is enough evidence.
- **When promoting a validated observation to a discipline**: add the rule to both `PROCESS.md` (Active Disciplines) and this file (`CLAUDE.md`), so it is enforced going forward.

Do not skip observation logging because it feels minor. Small frictions compound. The system only works if observations are captured in the moment.

## Key Files

- `NOTES.md` — institutional memory
- `PROCESS.md` — process improvement tracking (observations, disciplines)
