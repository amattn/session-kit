## NOTES.md Discipline

> **CRITICAL — This rule is non-negotiable and survives compaction.**
> If you remember nothing else from this file, remember this section.

**Update NOTES.md after every decision, before moving to the next topic.** This is not optional and not deferrable.

### Why this exists

The pattern of "I'll catch up on notes later" always fails. Decisions accumulate faster than memory, and after compaction the rationale is permanently lost. The cost of writing notes is seconds; the cost of lost rationale is re-litigating settled decisions in the next session.

### Rules

1. **Every decision gets a NOTES.md entry immediately** — before presenting the next item, before writing code, before doing anything else.
2. **If you realize you've fallen behind, stop everything and catch up.** Do not continue accumulating debt.
3. **Batch answers** (e.g., "1A, 2C, 3D") still get individual NOTES.md entries for each decision.
4. **After compaction or context recovery**, re-read NOTES.md to restore context. The file *is* your memory.

### Self-check

Before each response, ask yourself: "Did the user just make a decision? If yes, have I written it to NOTES.md yet?" If the answer is no, do that first.
